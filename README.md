1. ОПИСАНИЕ ПРОЕКТА
--------------------------------------------------------------------------------
ДипломРеестр — это веб-приложение и API для проверки подлинности дипломов о
высшем образовании. Платформа позволяет:

  - ВУЗам вести реестр выданных дипломов (добавление, аннулирование, удаление,
    загрузка через Excel/CSV, экспорт в Excel).
  - Студентам получать QR-коды и временные ссылки для подтверждения диплома с
    настраиваемым сроком действия (TTL).
  - Работодателям (HR) проверять дипломы через веб-интерфейс (по номеру диплома
    + код вуза, либо через загрузку QR-кода).
  - HR-порталам автоматически проверять дипломы через REST API (одиночная и
    массовая проверка).

Проект разработан на Python + Flask с использованием SQLite, поддерживает тёмную
тему, горячие клавиши, цифровую подпись дипломов (RSA) и защиту от
злоупотреблений (rate limiting, блокировка IP, отзыв QR-кодов).


2. ОСНОВНЫЕ ВОЗМОЖНОСТИ
--------------------------------------------------------------------------------
  - Управление дипломами: добавление по одному, массовая загрузка из Excel/CSV
    (колонки: ФИО | Год | Специальность | Номер диплома).
  - Аннулирование/восстановление: ВУЗ может изменить статус диплома
    (активен/аннулирован).
  - Удаление: удаление выбранного диплома (с выделением строки) или очистка
    всего списка.
  - Поиск и сортировка: фильтрация по ФИО, номеру диплома, году, специальности;
    сортировка по любому столбцу.
  - QR-код для студента: настраиваемый TTL (дни и часы), скачивание PNG.
  - Временная ссылка для студента: настраиваемый TTL (дни и часы),
    автоматическое копирование в буфер обмена.
  - Отзыв QR-кода: при создании нового QR-кода старый автоматически перестаёт
    действовать; также есть ручной отзыв.
  - Проверка диплома HR: по номеру диплома + выбор вуза из выпадающего списка;
    через загрузку QR-кода (распознавание OpenCV).
  - API для HR-порталов: одиночная проверка (POST /api/verify_diploma_for_hr)
    и массовая (POST /api/verify_diplomas_batch) с авторизацией по API-ключу.
  - Цифровая подпись: каждый диплом подписывается приватным ключом ВУЗа (RSA);
    при проверке автоматически отображается статус подписи.
  - Тёмная тема: переключение с сохранением выбора в localStorage.
  - Rate limiting: защита от брутфорса и DoS (настроены лимиты для каждого
    эндпоинта).
  - Блокировка IP: при многократном превышении лимитов IP блокируется на
    30-60 минут.
  - Логирование безопасности: запись подозрительных событий в таблицы БД.
  - Логирование проверок: фиксация всех обращений к дипломам (кто, когда,
    с какого IP, результат).
  - Хоткеи: ускорение работы с формами (Enter, Esc, Backspace).


3. ТЕХНОЛОГИЧЕСКИЙ СТЕК
--------------------------------------------------------------------------------
  - Язык: Python 3.9+
  - Веб-фреймворк: Flask 2.0+
  - База данных: SQLite (в планах переход на PostgreSQL)
  - Криптография: bcrypt (хеширование паролей), cryptography (RSA подписи)
  - QR-коды: qrcode (генерация), OpenCV (распознавание)
  - Excel/CSV: pandas + openpyxl
  - Rate limiting: flask-limiter
  - Токены: itsdangerous
  - Фронтенд: HTML5, CSS3, JavaScript (ванильный)

5. УСТАНОВКА И ЗАПУСК
--------------------------------------------------------------------------------
  5.1. Клонировать репозиторий или скачать архив.

  5.2. Создать виртуальное окружение (рекомендуется):
         python -m venv venv
         source venv/bin/activate      # Linux/macOS
         venv\Scripts\activate         # Windows

  5.3. Установить зависимости:
         pip install -r requirements.txt

       Если requirements.txt нет, установите вручную:
         pip install flask flask-limiter bcrypt pandas openpyxl qrcode
         pip install opencv-python itsdangerous python-dotenv cryptography

  5.4. Создать базу данных:
         python database/CREATEBD.py

  5.5. (Опционально) Добавить тестовые дипломы:
         python add_test_diplomas.py

  5.6. (Опционально) Сгенерировать RSA ключи для существующих ВУЗов:
         python generate_keys.py

  5.7. Настроить переменные окружения (для API HR-порталов):
         Создать файл .env в корне:
           HR_API_KEYS=hr_key_1,hr_key_2,hr_key_3

  5.8. Запустить сервер:
         python main.py

       После запуска откройте в браузере: http://127.0.0.1:5000

  5.9. Для управления вузами и просмотра логов используйте консольную утилиту:
         python university_admin.py

7. ИСПОЛЬЗОВАНИЕ ПЛАТФОРМЫ
--------------------------------------------------------------------------------
  7.1. Страница входа (выбор роли) - URL: /index
       - Выберите "Как студент" или "Как ВУЗ".
       - Для студента: введите ФИО и номер диплома.
       - Для ВУЗа: введите логин и пароль (задаются через university_admin.py).

  7.2. Личный кабинет ВУЗа (URL: /university)
       - Добавление диплома: заполнить поля (ФИО, год, специальность, номер)
         и нажать "Добавить". При добавлении автоматически вычисляется хеш
         и создаётся цифровая подпись (RSA).
       - Загрузка из Excel/CSV: кнопка "Загрузить из Excel/CSV" → выбрать файл.
         Формат: 4 колонки без заголовков (ФИО, Год, Специальность, Номер).
       - Таблица дипломов:
         * Сортировка: клик по стрелке в заголовке столбца.
         * Поиск: поле ввода (фильтрация по ФИО, номеру, году, специальности).
         * Выделение строки: клик по строке → активна кнопка "Удалить выбранное".
         * Аннулирование/восстановление: кнопка в столбце "Управление".
       - Экспорт в Excel: кнопка "Экспорт в Excel" → скачивается файл .xlsx
         с текущим отфильтрованным и отсортированным списком.
       - Очистка списка: удаляет все дипломы ВУЗа (с подтверждением).
       - Выход: кнопка в шапке или клавиша Esc.

  7.3. Личный кабинет студента (URL: /student)
       - Отображаются данные диплома (номер, год, специальность, код ВУЗа,
         статус).
       - Генерация QR-кода:
         * Выбрать дни и часы (TTL).
         * Нажать "Генерация QR-кода" → создаётся временная ссылка, QR-код
           отображается на странице.
         * Можно скачать PNG.
       - Отзыв QR-кода: кнопка "Отозвать QR-код" → активный токен удаляется,
         старый QR перестаёт работать.
       - Создание временной ссылки:
         * Выбрать дни и часы (TTL).
         * Нажать "Создать временную ссылку" → ссылка копируется в буфер
           обмена, действительна указанное время.
       - Выход: кнопка "Выйти из системы" или клавиша Esc.

  7.4. Проверка диплома (HR) (URL: /)
       - Режим "По номеру":
         * Выбрать ВУЗ из выпадающего списка (загружается из БД).
         * Ввести номер диплома.
         * Нажать "Проверить в реестре".
         * Результат: статус (подлинный/аннулирован/не найден), ФИО, год,
           специальность, а также информация о цифровой подписи.
       - Режим "Через QR":
         * Переключиться на "Через QR".
         * Загрузить изображение с QR-кодом (файл).
         * Нажать "Проверить в реестре" → система распознаёт QR (OpenCV),
           извлекает ссылку и проверяет диплом.
       - Горячие клавиши: Enter (проверить), Esc (сбросить форму).


8. API ДЛЯ HR-ПОРТАЛОВ
--------------------------------------------------------------------------------
  8.1. Одиночная проверка диплома
       - Эндпоинт: POST /api/verify_diploma_for_hr
       - Заголовки: Content-Type: application/json
       - Параметры (JSON):
           {
             "diploma_number": "ДП-2023-001",
             "university_code": 1,
             "api_key": "hr_key_1"          # опционально
           }
       - Ответ (успех, статус 200):
           {
             "status": "valid",
             "message": "Подлинный",
             "diploma_number": "ДП-2023-001",
             "university_code": 1,
             "university_name": "МГУ",
             "full_name": "Иванов Иван Иванович",
             "graduation_year": 2023,
             "specialty": "Программная инженерия",
             "verified_at": "2024-01-15T12:00:00"
           }
       - Коды ответа: 200 (найден), 404 (не найден), 401 (неверный ключ),
         429 (слишком много запросов).

  8.2. Массовая проверка (до 100 дипломов за запрос)
       - Эндпоинт: POST /api/verify_diplomas_batch
       - Параметры (JSON):
           {
             "diplomas": [
               {"diploma_number": "ДП-2023-001", "university_code": 1},
               {"diploma_number": "ДП-2023-002", "university_code": 1}
             ],
             "api_key": "hr_key_1"
           }
       - Ответ:
           {
             "results": [
               {"diploma_number": "ДП-2023-001", "status": "valid", "full_name": "..."},
               {"diploma_number": "ДП-2023-002", "status": "not_found"}
             ],
             "total": 2,
             "valid_count": 1,
             "invalid_count": 0,
             "not_found_count": 1,
             "verified_at": "2024-01-15T12:00:00"
           }

  8.3. Лимиты API
       - /api/verify_diploma_for_hr: 100 запросов в минуту, 1000 в час,
         10000 в день.
       - /api/verify_diplomas_batch: 10 запросов в минуту, 100 в час.

  8.4. Авторизация
       - API-ключ передаётся в поле "api_key". Ключи задаются в переменной
         окружения HR_API_KEYS (через .env). Если ключи не заданы, проверка
         отключается.


9. ЦИФРОВАЯ ПОДПИСЬ ДИПЛОМОВ
--------------------------------------------------------------------------------
  - Каждый диплом при добавлении (через форму) подписывается приватным ключом
    ВУЗа (RSA-2048). Подпись сохраняется в поле digital_signature.
  - При проверке диплома (через веб-интерфейс или API) автоматически
    верифицируется подпись с помощью публичного ключа ВУЗа.
  - Результат проверки отображается в ответе: поле "signature_valid" (bool) и
    "signature_message".
  - Если подпись отсутствует (старые дипломы или загруженные через Excel),
    выводится предупреждение.
  - Ключи хранятся в папке keys/university_<код>. Приватный ключ никогда не
    должен покидать сервер.


10. АДМИНИСТРИРОВАНИЕ И ЛОГИРОВАНИЕ
--------------------------------------------------------------------------------
  10.1. Консольная утилита (university_admin.py)
        - Запуск: python university_admin.py
        - Меню:
            1. Добавить вуз
            2. Удалить вуз
            3. Список вузов
            4. Логи безопасности (security_logs)
            5. Логи проверок дипломов (verification_logs)
            6. Выход

  10.2. Логи безопасности (security_logs)
        - Записываются подозрительные события: превышение лимитов запросов,
          неверные API-ключи, попытки брутфорса.
        - Поля: IP, эндпоинт, user-agent, время.

  10.3. Логи проверок дипломов (verification_logs)
        - Записывается КАЖДАЯ проверка диплома (успешная или нет).
        - Поля: ID диплома, код вуза, номер диплома, тип проверки
          (manual/temp_link/qr_scan), токен (если есть), IP, user-agent,
          результат, время.

  10.4. Просмотр логов через веб (опционально)
        - Эндпоинт /admin/logs?password=admin123 (доступен только с паролем).


11. БЕЗОПАСНОСТЬ
--------------------------------------------------------------------------------
  - Хеширование паролей: bcrypt.
  - Rate limiting: ограничение количества запросов по IP для каждого эндпоинта.
  - Блокировка IP: при многократном превышении лимитов IP блокируется на
    30–60 минут (таблица blocked_ips).
  - Отзыв QR-кодов: механизм active_token в БД, старые токены перестают
    работать при создании нового или ручном отзыве.
  - Цифровая подпись: защита от подделки данных диплома.
  - API-ключи для HR-порталов.
  - Логирование всех проверок для аудита.
  - Защита от CSRF через сессии Flask.
  - Экранирование HTML (escapeHtml) на фронтенде.


12. ПЛАНЫ РАЗВИТИЯ
--------------------------------------------------------------------------------
  - Переход с SQLite на PostgreSQL для работы с миллионами записей и высокими
    нагрузками.
  - Расширение API: вебхуки, OAuth-авторизация, получение полных данных
    диплома по токену студента.
  - Выход на национальный масштаб: географически распределённые серверы,
    балансировка нагрузки, отказоустойчивость.
  - Интеграция с мессенджером «Макс» для безопасной передачи паролей,
    уведомлений и обратной связи с вузами.
